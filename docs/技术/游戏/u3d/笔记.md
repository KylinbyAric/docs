# 组件

组件可以理解成一个对象的功能。比如mesh、材质、刚体、灯光等等。对象之间的区别就在于挂载的组件不一样。特殊的一点是所有的对象都有transform(位置组件)，因为这是物体存在于空间的标识。

# 脚本

## 生命周期

Awake：初始化时会调用，**只调用一次**。主要是进行对相见的初始化、单例等。
OnEnable：激活的时候会调用。每次对象变为激活时调用。
Start:首次update前会调用，只调用一次。
FixedUpdate：固定时间间隔调用，默认是0.02s。类似定时器
Update: 每一帧都会调用。常用的游戏逻辑处理。
LateUpdate：每一帧在所有脚本的Update方法调用之后调用。类似钩子函数。如摄像机的跟随等。
OnDisable:附着对象变为非激活状态时调用，主要用于注销、保存等操作。
OnDestroy:附着对象被销毁时调用，主要是进行最终清理。

### 激活和销毁的区别


| 维度             | 激活/禁用 (Active/Inactive)                             | 销毁 (Destroy)                              |
| :--------------- | :------------------------------------------------------ | :------------------------------------------ |
| **对象是否存在** | 对象**仍然存在于场景**中，只是被标记为不可见/不参与逻辑 | 对象**被从场景中彻底移除**，引用变为 `null` |
| **内存占用**     | **仍占用内存**（组件、数据都保留）                      | **释放内存**（除非有其他引用阻止垃圾回收）  |
| **能否恢复**     | **可以**，通过 `SetActive(true)` 立即恢复               | **无法恢复**，需重新实例化或从预制体生成    |
| **触发生命周期** | `OnEnable` / `OnDisable`                                | `OnDestroy`                                 |
| **对子对象影响** | 递归影响所有子对象                                      | 递归销毁所有子对象                          |
| **典型用途**     | 临时隐藏物体（如 UI 面板、子弹对象池）、暂停逻辑        | 永久删除物体（如敌人死亡、场景切换清理）    |

激活时
对象仍然存在，只是不可见。
仍占用内存
临时隐藏物体，UI面板、暂停等

销毁时
彻底移出场景
不再占用内存
敌人死亡/场景切换清理

## 执行顺序


| 维度                    | 规则 / 说明                                                                                                                               |
| :---------------------- | :---------------------------------------------------------------------------------------------------------------------------------------- |
| **整体阶段顺序**        | **固定**：`Awake` → `OnEnable` → `Start` → `Update` → `LateUpdate` → …（所有脚本先完成同一阶段，再进入下一阶段）                    |
| **同一阶段内（默认）**  | **不确定**，由 Unity 内部决定，无法预测谁先执行                                                                                           |
| **同一阶段内（可控）**  | 依**执行优先级数值从小到大**（升序）执行，数字越小越先执行                                                                                |
| **同一脚本的不同实例**  | **无法单独控制**顺序，只能控制不同脚本类之间的顺序                                                                                        |
| **设置方法① - 编辑器** | `Edit` → `Project Settings` → `Script Execution Order`，添加脚本并调整序号（数字越小越先）                                              |
| **设置方法② - 代码**   | 在脚本类上添加`[DefaultExecutionOrder(数值)]`，如 `[DefaultExecutionOrder(-100)]`                                                         |
| **编辑器与代码优先级**  | **编辑器面板中的设置优先**，会覆盖代码中的 `DefaultExecutionOrder`                                                                        |
| **未设置优先级的脚本**  | 全部在**Default Time** 槽位内以**不确定顺序**执行                                                                                         |
| **可靠设计建议**        | - 将不依赖他人的初始化放`Awake`，依赖他人的放 `Start`<br>- 使用集中式管理器手动控制初始化顺序<br>- 避免过度依赖脚本执行顺序来修正代码设计 |

整体阶段：
分阶段，统一执行。先执行所有脚本的Awake，然后再是OnEnable。逐次顺序执行。

同一阶段：
默认是随机执行。
设置优先级的情况下则是数字小的先执行。

同一脚本的不同实例：无法确定顺序

| **能否动态修改**     | 可以（运行时通过`tag = "NewTag"` 赋值，但有性能损耗且容易出错，不推荐）                     | 可以（运行时通过`layer = 3` 或 `LayerMask.NameToLayer` 赋值）                            |

## 向量

### 定义

标量：只有数值
向量：有数值，也有方向。
向量的模：向量的大小
单位化/归一化：将向量转为单位向量的过程

### 运算

加法：A(x1,y2) + B(x2,y2) = (x1+x2,y1+y2)。几何意义上就是两个向量构建的平行四边形的对角线。
减法：A(x1,y2) - B(x2,y2) = (x1-x2,y1-y2)。几何意义上就是A的终点到B的终点的连线(平行四边形短的对角线)
乘法：A(x1,y2) * 2 = (x1 * 2,y1 * 2)

#### 一、点乘的数学定义

**点乘（Dot Product）** 是线性代数中两个向量之间的一种运算，结果是一个**标量**（一个数字）。

* **公式**（分量形式）：
  对于向量 `A = (x1, y1, z1)` 和 `B = (x2, y2, z2)`
  `A · B = x1*x2 + y1*y2 + z1*z2`
* **公式**（几何形式）：
  `A · B = |A| * |B| * cosθ`
  其中 `|A|`、`|B|` 是向量的长度（模），`θ` 是 A 与 B 之间的夹角（0\~π）。

#### 二、几何意义 —— 方向关系的“标尺”

从 `cosθ` 的符号和大小可以直观判断两个向量的方向关系：


| `A·B` 的值 | 对应的几何关系             | 夹角 θ            |
| ----------- | -------------------------- | ------------------ |
| **正数**    | 方向大致相同（指向同一侧） | 0° ≤ θ < 90°   |
| **0**       | 互相**垂直**（正交）       | θ = 90°          |
| **负数**    | 方向大致相反（指向相反侧） | 90° < θ ≤ 180° |
| `           | A                          |                    |
| `-          | A                          |                    |

> 当 A 和 B 都是**单位向量**（长度为1）时，`A·B = cosθ`，直接得到夹角的余弦值。

#### 三、在图形编程中的核心意义

点乘在 3D 图形学、游戏开发（Unity/Unreal）、着色器编程中无处不在，主要用来判断方向、角度和投影。

##### 🔹 1. 光照计算（漫反射光照 - Lambert）

* **公式**：`diffuse = max(0, normal · lightDir)`
* 用法：表面法线 `normal` 点乘 光源方向 `lightDir`。结果 >0 表示表面朝向光源，结果越大光照越强；结果为 0 或负值则无光。
* 这是所有实时渲染的基础。

##### 🔹 2. 判断物体的前后关系（视锥/视线）

* 用法：摄像机前向向量 `cameraForward` 点乘 从摄像机指向物体的向量 `dirToObject`。
  * 结果 >0 → 物体在摄像机**前方**。
  * 结果 <0 → 物体在**后方**（可能被剔除）。
* 类似用途：AI 视野检测（前方扇形范围）。

##### 🔹 3. 计算两个向量之间的夹角

* 用法：`θ = arccos( (A·B) / (|A||B|) )`
* 用途：计算转向角度、子弹命中角度、判断是否朝向目标等。

##### 🔹 4. 计算投影长度（投影）

* 用法：`projLength = A · (B / |B|)` 或直接用单位向量。
* 用途：例如将一个点的速度分解到某个轴的方向上。

##### 🔹 5. 镜面高光（Blinn-Phong 简化）

* 用法：半角向量 `H` 点乘法线 `N`，结果用于控制高光强度。

##### 🔹 6. 判定方向相似度

* 用法：例如 Steering Behavior 中，期望方向点乘当前方向，结果越接近 1 表示越一致。

#### 一、叉乘的数学定义

**叉乘（Cross Product）** 是只在三维空间中定义的向量运算，结果是一个**新的向量**，该向量垂直于原来的两个输入向量。

* **公式**（分量形式）：
  对于向量 `A = (x1, y1, z1)` 和 `B = (x2, y2, z2)`
  **text**

  ```
  A × B = ( y1*z2 - z1*y2 ,
            z1*x2 - x1*z2 ,
            x1*y2 - y1*x2 )
  ```
* **公式**（几何形式）：
  `|A × B| = |A| * |B| * sinθ`
  其中 `θ` 是 A 与 B 之间的夹角（0\~π）。

#### 二、几何意义 —— 生成垂直向量与有向面积

叉乘的结果向量 `C = A × B` 具有以下性质：


| 属性         | 含义                                                                                                                 |
| ------------ | -------------------------------------------------------------------------------------------------------------------- |
| **方向**     | 垂直于`A` 和 `B` 所张成的平面，遵循**右手定则**：<br/>四指从 `A` 弯向 `B`（角度 < 180°），拇指方向即为 `C` 的方向。 |
| **长度**     | `                                                                                                                    |
| **交换律**   | `A × B = - (B × A)`，叉乘反交换。                                                                                  |
| **特殊情况** | 若`A` 与 `B` 平行（`sinθ = 0`），则 `A × B = 0`（零向量）。                                                        |

> 右手定则图示：
>
> * 右手坐标系（Unity 世界坐标）：X右，Y上，Z前。
> * `X × Y = Z`，`Y × Z = X`，`Z × X = Y`。

#### 三、在图形编程中的核心意义

叉乘在 3D 图形学、游戏开发、物理引擎中主要用于**计算方向、法线、旋转轴和力矩**。

##### 🔹 1. 计算垂直向量（法线）

* **用法**：已知一个三角形的两条边向量 `AB` 和 `AC`，通过 `AB × AC`（或 `AC × AB`，取决于朝向）得到该三角形的**法线向量**。
* **应用**：生成网格的法线数据，用于光照计算；计算平面方程。

##### 🔹 2. 构建坐标系（基向量）

* **用法**：已知一个方向（例如前向 `forward`）和另一个参考方向（例如世界向上 `up`），通过叉乘补齐完整的局部坐标系的三个轴。
  * `right = up × forward`（归一化后）
  * `trueUp = forward × right`
* **典型场景**：摄像机跟随、导弹制导、`Transform.LookAt` 的内部实现。

##### 🔹 3. 判断方向性（左/右）

* **用法**：对于 `forward` 和 `toTarget`，计算 `cross = forward × toTarget`，然后检查 `cross.y` 的符号（在水平面上）：
  * `cross.y > 0` → 目标在**右侧**。
  * `cross.y < 0` → 目标在**左侧**。
* **应用**：绕 Y 轴旋转的转向判断、UI 箭头指示器。

##### 🔹 4. 计算力矩与角速度（物理）

* **用法**：力矩 `τ = r × F`，其中 `r` 是力臂向量，`F` 是力向量。叉乘结果的大小等于力矩大小，方向是旋转轴方向。
* **应用**：刚体受力产生旋转的物理模拟。

##### 🔹 5. 计算旋转轴（从一个向量旋转到另一个）

* **用法**：旋转轴 = `A × B`（归一化）。`A` 旋转到 `B` 的最短路径旋转轴垂直于两者平面。
* **应用**：`Quaternion.FromToRotation` 的原理，实现物体转向动画。

##### 🔹 6. 判断点是否在三角形内部（重心坐标辅助）

* 通过边缘叉乘符号一致性判断。

#### 插值


##### 一、插值的数学定义

插值的基本形式：给定起点 A**A** 和终点 B**B**，以及一个插值因子 t**t**（通常 0≤t≤1**0**≤**t**≤**1**），计算中间值 P**P**。

P=f(A,B,t)**P**=**f**(**A**,**B**,**t**)当 t=0**t**=**0** 时 P=A**P**=**A**；t=1**t**=**1** 时 P=B**P**=**B**；t=0.5**t**=**0.5** 时 P**P** 在中间位置。

最常见的插值类型是**线性插值（Linear Interpolation, Lerp）**：

* **标量（浮点数）**：
  Lerp(a,b,t)=a+(b−a)×t=(1−t)⋅a+t⋅b**Lerp**(**a**,**b**,**t**)**=**a**+**(**b**−**a**)**×**t**=**(**1**−**t**)**⋅**a**+**t**⋅**b
* **向量**：对每个分量分别线性插值。
  Lerp(A⃗,B⃗,t)=A⃗+(B⃗−A⃗)×t**Lerp**(**A**,**B**,**t**)**=**A**+**(**B**−**A**)**×**t
* **颜色**（RGBA）：对红、绿、蓝、透明度各通道分别线性插值。

##### 二、图形编程中的常见插值类型


| 类型               | 英文         | 数学特征                | 典型用途                               |
| ------------------ | ------------ | ----------------------- | -------------------------------------- |
| **线性插值**       | Lerp         | 匀速直线运动            | 简单移动、褪色、进度条                 |
| **球面线性插值**   | Slerp        | 沿球面最短弧插值        | 旋转（四元数）、摄像机朝向             |
| **平滑插值**       | SmoothStep   | 曲线先快后慢再快（S形） | 自然淡入淡出、摄像机缓动               |
| **三次贝塞尔插值** | Cubic Bezier | 自定义曲线形状          | 动画曲线、材质编辑器的动画曲线         |
| **指数/平滑阻尼**  | SmoothDamp   | 近似物理弹簧阻尼        | 平滑跟随、摄像机平滑跟随（Unity 内置） |

> **注**：对于四元数（Quaternion）旋转，必须用 **Slerp** 而非 Lerp，否则旋转路径会扭曲。

##### 三、在游戏开发中的核心应用

###### 🔹 1. 物体平滑移动

不用 `transform.position = targetPosition`（瞬移），而用 Lerp 每帧靠近一小段，产生跟随或缓动效果。

**csharp**

```
transform.position = Vector3.Lerp(transform.position, targetPosition, Time.deltaTime * speed);
```

###### 🔹 2. 颜色渐变

**csharp**

```
renderer.material.color = Color.Lerp(startColor, endColor, t);
```

###### 🔹 3. 摄像机平滑跟随

使用 `Vector3.SmoothDamp` 或 `Mathf.SmoothDamp` 实现带惯性效果的跟随，避免突兀。

###### 🔹 4. 旋转插值（四元数）

**csharp**

```
transform.rotation = Quaternion.Slerp(transform.rotation, targetRotation, Time.deltaTime * rotateSpeed);
```

###### 🔹 5. 动画混合

角色动画中，根据速度参数在 Idle 和 Run 动画之间插值权重；或使用 `Mathf.SmoothStep` 控制 UI 面板的显示/隐藏动画。

###### 🔹 6. 纹理采样与抗锯齿

GPU 在采样纹理时，根据 UV 坐标在两个相邻像素颜色之间做双线性插值（2D Lerp），使图像放大后不平滑缺失。
